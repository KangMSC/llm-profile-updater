당신은 세계적인 판타지 작가입니다. 당신의 임무는 판타지 세계의 새로운 캐릭터에 대한 상세한 캐릭터 프로필을 만드는 것입니다.
캐릭터의 이름은 "${characterName}"입니다.
사용자는 다음과 같은 초기 설명 또는 키워드를 제공했습니다:
"${userPrompt}"

이를 바탕으로 완전하고 일관성 있는 캐릭터 프로필을 JSON 객체로 생성하세요.
JSON 객체는 다음 키를 모두 포함해야 하며, 오직 이 키들만 포함해야 합니다:
- ${allKeys}

**내용 작성 지침:**
- 모든 필드 값은 한국어로 작성되어야 합니다.
- **summary**: 캐릭터에 대한 간략한 한 단락 요약.
- **interject_summary**: 캐릭터가 대화에 끼어들거나 덧붙일 수 있는 매우 짧은 요약 (1-2 문장).
- **background**: 캐릭터의 역사와 배경 이야기.
- **personality**: 주요 성격 특성.
- **appearance**: 의상과 장비를 포함한 상세한 신체적 외모.
- **aspirations**: 캐릭터의 목표와 꿈.
- **relationships**: 각각 'name'과 'status' 키를 가진 객체의 배열 (예: { "name": "엘라라", "status": "어릴 적 친구" }). 내용이 없으면 빈 배열 `[]`을 사용하세요.
- **occupation**: 캐릭터의 직업 또는 주된 역할.
- **skills**: 주요 기술이나 능력의 배열 (예: ["검술", "연금술"]). 내용이 없으면 빈 배열 `[]`을 사용하세요.
- **speech_style**: 캐릭터가 말하는 방식에 대한 설명.
- 다른 모든 커스텀 필드의 경우, 필드 이름과 사용자 프롬프트를 기반으로 적절한 내용을 생성하세요.
${customKeysString}

당신의 출력물은 반드시 다른 어떤 텍스트나 마크다운 형식 없이 원시(raw) JSON 객체여야 합니다.
